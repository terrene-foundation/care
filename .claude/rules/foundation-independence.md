# Foundation Independence

## Scope

These rules apply to ALL content and operations across the Terrene Foundation ecosystem.

## The Principle

The Terrene Foundation is a **sovereign, independent entity** governed by its constitution. There is NO structural relationship between the Foundation and any commercial entity.

## Rules

### 1. No Commercial Coupling

Content across all repos MUST NOT imply, suggest, or create coupling between the Foundation and any commercial entity. The Foundation publishes open standards and SDKs — commercial entities build on them. That is the intended model.

### 2. IP Ownership

- Specifications (CARE, EATP, CO, PACT, CDI): **CC BY 4.0** — Foundation-owned
- Open source code (Kailash SDK, EATP SDK): **Apache 2.0** — Foundation-owned
- All open-source IP: **fully and irrevocably transferred** to the Foundation
- BSL 1.1 is **NOT** open source — use "source-available" or "open-core"

### 3. Canonical Naming

- The Foundation is **Terrene Foundation** (not OCEAN Foundation)
- Domain: `terrene.foundation`
- Developer portal: `terrene.dev`
- GitHub org: `terrene-foundation`

### 4. Derivation Pathway

External-facing content MUST lead with the discovery, not the mission. COC came first; governance specifications followed. The full derivation pathway rules are owned by foundation/ (its repo governs its own content).

### 5. Governance Layer Positioning

The Foundation complements execution tools — it does not compete with them. Vendor-neutral language always. The full positioning rules are owned by foundation/ (its repo governs its own content).

### 6. Child Repo Governance Sovereignty

This orchestration layer references principles from child repos but does NOT own them. Each child repo's `.claude/rules/` are authoritative for their own content. When orchestrating across repos, defer to the child repo's rules for domain-specific decisions.
