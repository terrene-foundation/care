# Terrene Foundation Naming

## Scope

These rules apply to ALL new content and edits in this knowledge base.

## Rules

### Foundation Name

The Foundation is **Terrene Foundation** (formerly OCEAN Foundation).

- Domain: `terrene.foundation`
- Developer portal: `terrene.dev`
- GitHub org: `terrene-foundation`
- When writing new content, always use "Terrene Foundation"
- When editing an existing file for other reasons, update "OCEAN Foundation" to "Terrene Foundation"
- Existing docs may still say "OCEAN" — known transition, not an error unless you're editing the file

### License Accuracy

- Specifications (CARE, PACT, EATP, CO, CDI): **CC BY 4.0** (NOT CC-BY-SA — ShareAlike would prevent proprietary implementations)
- Open source code (Kailash Python, EATP SDK, CO Toolkit): **Apache 2.0** — Foundation-owned
- All open-source IP: **fully and irrevocably transferred** to the Foundation
- BSL 1.1 is **NOT** open source — use "source-available" or "open-core"

### Foundation Independence

The Foundation is a **sovereign, independent entity** governed by its constitution. There is NO structural relationship between the Foundation and any commercial entity.

- **Foundation**: Specs (CC BY 4.0) + open SDKs (Apache 2.0) — non-profit, neutral convener
- All open-source IP is Foundation-owned (fully transferred, irrevocable)
- Contributors (including the Founder) contribute as **individuals**, governed by the constitution
- The constitution prevents open-washing, rent-seeking, and self-interest by ANY party
- Commercial entities may build products on Foundation standards — that is the intended model
- Do NOT describe any contributor or company as having a "relationship" with or "arrangement" with the Foundation
- Do NOT frame open-source IP as "donated by," "licensed from," or "contributed by" any company
- Kailash Python, EATP SDK, and all Foundation code are **Foundation-owned**, full stop

### Canonical Terminology

- CARE planes: **Trust Plane** + **Execution Plane** (NOT operational/governance plane)
- Constraint dimensions: **Financial, Operational, Temporal, Data Access, Communication**
- CO = Cognitive Orchestration (domain-agnostic base methodology)
- COC = Cognitive Orchestration for Codegen (CO applied to software development — the "C" already means "for Codegen")
- PACT = Principled Architecture for Constrained Trust (**Working Architecture**, not yet a standard)
- The Trinity: CARE (philosophy) + EATP (protocol) + CO (methodology) — these are the standards
- PACT is a Working Architecture that provides organizational structure for the Trinity standards
- PACT specifies: D/T/R grammar, Operating Envelope delegation, Knowledge Clearance, Verification Gradient, Positional Addressing

### CO as Methodology

The Foundation publishes CO as an open methodology under CC BY 4.0. The Foundation does NOT sell methodology consulting. Publishing open methodologies is consistent with the Foundation mandate.
